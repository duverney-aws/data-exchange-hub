# Project Memory: Pharma Data Exchange Hub

## Project Overview
AWS Solutions Architecture engagement with Merck to develop a CMO (Contract Manufacturing Organization) data exchange platform that reduces integration time from 3-6 months to 1-4 weeks.

## Business Problem
- Current CMO integration takes 3-6 months due to legal, technical, and trust barriers
- High cost (200+ hours per CMO)
- Non-scalable custom integrations
- Need to handle both structured AND unstructured data

## Solution: Pharma Data Exchange Hub
3 proven integration patterns + AI processing + unified governance

### Three Integration Patterns
1. **Native Platform Connectors** - Cloud data warehouses (Snowflake, Databricks, BigQuery), databases (Oracle, SQL Server, MySQL), enterprise apps (SAP)
2. **Secure File Transfer** - Legacy/on-prem via SFTP, universal fallback for ANY system
3. **Unstructured Data AI** - Documents (Textract), images (Rekognition), IoT (IoT Core)

### Key AWS Services
- **Integration:** Transfer Family, Glue, AppFlow
- **AI/ML:** Textract, Rekognition, Comprehend, Bedrock (Gen AI)
- **Storage:** S3, Timestream, OpenSearch
- **Security:** KMS, Macie, CloudTrail, Audit Manager
- **Governance:** Lake Formation, Glue Data Quality

### Three Accelerators
1. Pre-negotiated legal templates (4-8 weeks → <1 week)
2. Self-service CMO onboarding (no Merck IT bottleneck)
3. Pre-built pharma data models (structured + unstructured)

## Key Decisions Made

### Decision 1: Simplified to 3 Patterns
- **Issue:** AWS Data Exchange retiring, Clean Rooms not a customer requirement
- **Resolution:** Removed Patterns 1 & 2, renumbered remaining patterns
- **Rationale:** Simpler to explain, faster to implement, covers 100% of CMO scenarios
- **Impact:** Native Connectors + SFTP cover all CMOs; AI capabilities unchanged

### Decision 2: Platform-Agnostic Approach
- **Issue:** Original docs were Snowflake-focused
- **Resolution:** Rewrote Pattern 1 (formerly 3) to be generic, supporting ANY platform
- **Rationale:** CMOs use diverse platforms (Snowflake, Oracle, SAP, etc.)

### Decision 3: Unstructured Data as Core Capability
- **Issue:** Initial focus was structured data only
- **Resolution:** Added Pattern 3 for documents, images, IoT
- **Rationale:** CMOs produce PDFs, visual inspection images, sensor data

### Decision 4: AI-First Design
- **Issue:** Traditional data integration approach
- **Resolution:** Integrated Textract, Rekognition, Bedrock throughout
- **Rationale:** 80% automation in document processing, Gen AI for insights
