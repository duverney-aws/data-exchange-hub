Step-by-Step Walkthrough

CMO Self-Service Experience

1 CMO Registration
  • CMO receives invitation email from Merck
  • Clicks unique onboarding URL
  • Creates account (name, email, company)
  • Sets up authentication (password, MFA)
Time: 5 minutes
2 Pattern Selection
  • Portal shows 3 integration patterns
  • CMO selects based on their capabilities:
    • Pattern 1: Native Connector (Snowflake, Oracle, SAP)
    • Pattern 2: SFTP (works with any system)
    • Pattern 3: Unstructured AI (PDFs, images)
  • Portal provides pattern-specific instructions
Time: 3 minutes
3 Schema Definition
  • CMO uploads sample data file OR
  • Selects from standard templates (batch, quality, materials)
  • Portal auto-infers schema from sample
  • CMO reviews/edits schema in visual editor
Time: 5 minutes

Field Mapping

CMO Fields → Merck Canonical Model

4 Field Mapping (Drag & Drop)

CMO maps their field names to Merck's canonical data model

Example Mapping

CMO Field Name CMO Data Type → Merck Canonical Field
lot_number string → batch_id
prod_code string → product_code
start_dt datetime → manufacturing_start_date
end_dt datetime → manufacturing_end_date
yield_pct decimal → yield_percentage
status string → quality_status
Smart Mapping: Portal suggests mappings based on field names and data types. CMO just confirms or adjusts.

Time: 5 minutes

Test & Validate

Real-Time Validation Feedback

5 Upload Test Data
  • CMO uploads test file (CSV, JSON, Parquet)
  • Portal validates against defined schema
  • Real-time feedback on errors
  • Data quality score calculated

Validation Results Example

✅ Validation Passed (Score: 98/100)

  • ✅ All required fields present
  • ✅ Data types correct
  • ✅ No duplicate batch IDs
  • ✅ Yield percentages in valid range (0-100)
  • ⚠️ 2 records missing optional field "equipment_id"

Status: Ready for production

If Validation Fails

❌ Validation Failed (Score: 65/100)

  • ❌ 5 records missing required field "batch_id"
  • ❌ 3 records have invalid date format
  • ⚠️ 10 records have yield > 100% (check data)

Action: Fix errors and re-upload

Time: 10 minutes (including fixes)

What Happens Behind the Scenes

When Merck Clicks "Activate"

Step Functions Workflow (Fully Automated): 1. Create S3 bucket prefix for CMO s3://merck-cmo-hub/bronze/cmo-alpha/ 2. Register schema in Glue Schema Registry Schema: batch-manufacturing v1.0 3. Store data contract in DynamoDB Contract ID: dc-12345 4. Deploy Glue ETL job from template Job: cmo-alpha-batch-manufacturing-etl • Inject schema from registry • Inject field mappings from contract • Inject data quality rules 5. Configure connection (pattern-specific) • Pattern 1: Glue connection to CMO database • Pattern 2: Transfer Family SFTP user • Pattern 3: S3 presigned upload URL 6. Setup monitoring dashboard CloudWatch dashboard: cmo-alpha-monitoring 7. Send confirmation email to CMO Subject: "Your pipeline is ready!" Body: Connection details, documentation link TOTAL TIME: < 5 minutes
Zero Manual Work: Everything is automated via Infrastructure as Code (IaC)
