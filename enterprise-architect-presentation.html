Data Contract Architecture

Formal Agreements for Data Movement

What is a Data Contract?

A formal, machine-readable agreement between CMO (producer) and Merck (consumer)

Example Data Contract (YAML)

data_contract: version: "1.0" cmo_id: "cmo-alpha" data_domain: "batch-manufacturing" schema: fields: - name: batch_id type: string required: true unique: true - name: yield_percentage type: decimal min: 0 max: 100 data_quality: completeness: 100% uniqueness: [batch_id] timeliness: "< 24 hours" delivery: frequency: "daily" schedule: "02:00 UTC" format: "parquet" mapping: lot_number: batch_id yield_pct: yield_percentage

Benefits

  • Clarity: Both parties know expectations
  • Automation: Machine-readable enables validation
  • Versioning: Track changes over time
  • Governance: Centralized repository
  • Accountability: SLAs define success

Schema Registry & Validation

AWS Glue Schema Registry

Centralized Schema Management

GLUE SCHEMA REGISTRY ├── batch-manufacturing │ ├── v1.0 (2024-01-15) - Initial │ ├── v1.1 (2024-03-20) - Added yield_percentage │ └── v2.0 (2024-06-10) - Breaking change ├── quality-control │ ├── v1.0 (2024-01-15) - Initial │ └── v1.1 (2024-04-01) - Added test_method └── materials └── v1.0 (2024-01-15) - Initial

Validation Flow

  1. Data arrives in Bronze layer
  2. Lambda retrieves schema from registry
  3. Validates structure, types, constraints
  4. Applies data quality rules from contract
  5. Calculates quality score (0-100)
  6. If valid (>95%) → Silver layer
  7. If invalid (<95%) → Quarantine + alert CMO

Schema Evolution

Change Type Version Compatible?
Add optional field Minor (1.0 → 1.1) Yes
Add required field Major (1.0 → 2.0) No
Rename field Major (1.0 → 2.0) No

Architecture Principles

  • Decoupled: Hub isolates CMOs from Merck systems
  • Canonical model: Consistent regardless of source
  • Multi-tenancy: Isolated per CMO, shared infrastructure
  • Event-driven: Real-time and batch processing
  • Immutable: Bronze layer preserves audit trail

Three Integration Patterns

100% CMO Coverage

Pattern Best For Time Technology
1. Native Connectors Modern platforms (Snowflake, Oracle, SAP) 1-2 weeks AWS Glue (20+ connectors)
2. Secure Transfer Any system (universal fallback) 2-4 weeks AWS Transfer Family (SFTP)
3. Unstructured AI Documents, images, IoT 1-2 weeks Textract, Rekognition, IoT Core

Pattern Selection Guide

Decision Flow:
  1. Does CMO have modern data platform? → Pattern 1
  2. Legacy or on-prem system? → Pattern 2
  3. Unstructured data (PDFs, images)? → Add Pattern 3

Common Combinations

  • Cloud-native CMO: Pattern 1 + Pattern 3
  • Enterprise database: Pattern 1 + Pattern 3
  • Legacy system: Pattern 2 + Pattern 3
Key Point: Pattern 2 (SFTP) works with 100% of CMOs as universal fallback

Data Lake Architecture

Medallion: Bronze → Silver → Gold

Three-Layer Architecture

Bronze Layer (Raw / Immutable)
  • Exact copy of source data
  • No transformation
  • S3 Object Lock (WORM compliance)
  • 7-year retention (GxP)
Silver Layer (Validated / Standardized)
  • Canonical pharma data model
  • Data quality checks applied
  • Schema enforcement
  • Parquet format (columnar, compressed)
Gold Layer (Business-Ready / Aggregated)
  • Pre-aggregated metrics
  • Cross-CMO analytics
  • Optimized for BI tools

Canonical Data Domains

  • batch-manufacturing: Batch ID, product, dates, yields
  • quality-control: Test results, specifications, deviations
  • materials: Raw materials, lot numbers, suppliers
  • equipment: Equipment ID, maintenance, calibration
  • environmental: Temp, humidity, pressure, cleanroom
  • documents: CoA, batch records, deviations, CAPAs

Security Architecture

Defense-in-Depth, GxP-Ready

Multi-Layer Security

Layer Controls
Network VPC, private subnets, VPC endpoints, security groups
Identity IAM Identity Center (SSO), IAM roles, ABAC, MFA
Data Protection KMS encryption (per-CMO keys), S3 Object Lock, Macie
Governance Lake Formation (column/row access), Glue Data Catalog
Monitoring CloudTrail, CloudWatch, Security Hub, GuardDuty
Compliance AWS Audit Manager (21 CFR Part 11)

Per-CMO Isolation

  • S3 bucket prefixes: s3://bucket/cmo-alpha/
  • Lake Formation: Column/row-level permissions
  • IAM policies: Attribute-based access control (ABAC)
  • KMS keys: Optional per-CMO encryption keys
  • CloudTrail: All access logged and auditable

Compliance Alignment

  • 21 CFR Part 11 (electronic records/signatures)
  • GxP (Good Manufacturing Practice)
  • GDPR (if EU CMOs involved)
  • SOC 2 Type II, ISO 27001 (AWS certified)

Scalability & Performance

Serverless Auto-Scaling

Performance Benchmarks

Workload Volume Target Service
Batch ingestion 1 TB/day per CMO < 2 hours Glue (50 DPUs)
File processing 1000 files/hour < 5 min/file Lambda + Glue
SQL queries 100 GB scan < 10 seconds Athena (Parquet)
Document AI 1000 pages/hour < 30 sec/doc Textract async
Gen AI queries 100 queries/min < 3 seconds Bedrock (Claude)

Scalability Features

  • Serverless compute: Auto-scaling (0 to 1000+ concurrent)
  • Unlimited storage: S3 scales automatically
  • Parallel processing: Glue auto-scales workers
  • Columnar format: Parquet = 10x faster queries
  • Partition pruning: Query only relevant data

Cost Optimization

  • S3 Intelligent-Tiering (auto-move to cheaper storage)
  • Glue job bookmarks (incremental processing only)
  • Athena result caching (avoid re-scanning)
  • Spot instances for non-critical jobs (70% savings)
