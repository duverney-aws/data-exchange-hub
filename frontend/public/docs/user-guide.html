Overview

The Pharma Data Exchange Hub is a self-service platform that enables pharmaceutical companies (Merck) to securely receive manufacturing data from Contract Manufacturing Organizations (CMOs). It reduces CMO onboarding time from 3–6 months to 1–4 weeks through standardized integration patterns, automated schema validation, and SLA compliance tracking.

Key Outcomes: 90% reduction in integration time • 90% reduction in Merck effort per CMO • 100% CMO coverage through 3 flexible patterns • 80% automation in document processing • Natural language queries across all CMO data

Complete Onboarding Workflow

The correct sequence for adding a new CMO to the platform:

Step 1: Register CMO [Merck Admin] Step 2: Create Product(s) [Merck Admin] Step 3: Create Connection(s) [Merck Admin or CMO] Step 4: Activate Connection(s) [Merck Admin] Step 5: Create / Infer Schema(s) [Merck Admin] Step 6: Register Schema in Glue [Merck Admin] Step 7: Create Data Contract [Merck Admin] Step 8: Submit Contract to CMO [Merck Admin] Step 9: Invite CMO User [Merck Admin] Step 10: CMO Reviews & Accepts Contract [CMO User] Step 11: Merck Approves → Active [Merck Admin] Step 12: CMO Creates Batches & Submits [CMO User] Step 13: Monitor SLA Compliance [Merck Admin]

Three Integration Patterns

Every CMO can use one or more of these patterns depending on their technical capabilities:

Pattern 1: Native Database Connectors

AspectDetail
Use CaseCMOs with modern databases (Oracle, SQL Server, PostgreSQL, MySQL, Snowflake, SAP HANA)
How It WorksCMO submits DB credentials → Merck reviews → Merck activates → AWS Glue Connection + ETL Job created
Data FlowGlue ETL Job runs on daily schedule (2am UTC) → extracts data via JDBC → writes Parquet to Bronze S3 layer
ConnectivityDirect (public/VPN), NLB (Network Load Balancer), or AWS PrivateLink (most secure)
Multi-TableOne connection can have multiple table mappings — each gets its own Glue job and data domain
CMO EffortFill in a form with DB details + table mappings. No AWS account needed.

Pattern 2: Secure File Transfer (SFTP)

AspectDetail
Use CaseCMOs with legacy systems, on-premise infrastructure, or no cloud — universal fallback (100% coverage)
How It WorksMerck activates SFTP → credentials generated → CMO uses any SFTP client (FileZilla, WinSCP, CLI)
Data FlowCMO uploads file → lands in S3 incoming → Lambda processes → moved to date+lot partitioned Bronze path
Supported FormatsCSV, JSON, Parquet (other extensions stay in incoming/)
Upload Path/{batchId}/{elementType}/{filename} — automatically tags files to the correct batch and element
CMO EffortConnect via standard SFTP client with provided credentials. No AWS account or SDK needed.

Pattern 3: AI-Powered Unstructured Data

AspectDetail
Use CaseCMOs submitting PDFs (Certificates of Analysis, batch records) or images (defect photos, labels)
How It WorksCMO uploads document from Batch Detail page → AI extracts structured data → results saved to Bronze
Data FlowUpload → S3 → Textract (PDFs, multi-page) or Rekognition (images) → JSON result in Bronze layer
Confidence Routing≥85% confidence → auto-processed to Bronze layer | <85% → routed to manual-review prefix
Supported FormatsPDF (.pdf), Images (.png, .jpg, .jpeg, .tiff)
CMO EffortClick "Upload" on Batch Detail page and select a file. Results viewable immediately in the portal.

S3 Data Lake Path Convention

All ingested data lands in the Bronze layer with a consistent, queryable path structure:

bronze/{cmoId}/{dataDomain}/lot={lotNumber}/year=YYYY/month=MM/day=DD/{filename}

The lot= partition uses the CMO's own lot number — the regulated identifier that appears on physical documents and in Merck's QMS. This enables direct Athena queries filtered by lot number without any joins.

Connections

Connection Lifecycle

pending → (CMO configures, for native connectors) → pending_merck_review → (Merck activates) → active

Merck Admin Actions

  • Create Connection — choose CMO + type (SFTP, AI, or Native Connector)
  • Activate — provisions credentials (SFTP: Secrets Manager username/password; Native: Glue Connection + ETL Jobs; AI: configures upload endpoint)
  • Review & Activate — for native connectors submitted by the CMO (reviews DB details, one-click activate)

CMO User Actions

  • View My Connections — see all active connections with credentials and status
  • Add Database Connection — self-service form for native connector (DB type, host, port, credentials, table mappings, PrivateLink)

Security Properties

  • SFTP passwords: generated randomly (24 chars), stored in AWS Secrets Manager only
  • Database credentials: CMO enters directly in portal → stored in Secrets Manager → never returned to frontend after initial save
  • Merck admin review: sees host/port/database/username but never the password
  • PrivateLink: traffic stays on AWS backbone, no public internet traversal

Schema Management

Schemas define the expected structure of data from each CMO. They are stored in DynamoDB and registered in AWS Glue Schema Registry for validation.

Schema Lifecycle

Create (draft) → Register in Glue (registered) → Reference from Contract ↑ Edit (resets to draft if previously registered)

Two Ways to Create a Schema

MethodHowWhen to Use
Manual DefinitionAdd fields one by one (name, type, required/optional)You know the exact schema you want
File InferenceUpload a sample CSV or JSON → system auto-detects fields and typesYou have a sample file and want to start quickly

Supported Field Types

string, number, integer, boolean, date, timestamp, array, object

Batch Management

A batch represents a physical manufacturing event — identified by the CMO's lot number, linked to a product and a CMO. Each batch requires 4 data elements to be complete:

ElementDescriptionTypical SLA
BMR (Batch Manufacturing Record)Complete manufacturing process record5 days after manufacturing
CoA (Certificate of Analysis)Quality testing results and compliance certification5 days after manufacturing
In-Process ControlsMeasurements taken during manufacturing1–2 days after manufacturing
Yield DataProduction output quantities and efficiency metrics5 days after manufacturing

Batch Status Flow

in_progress → (all 4 elements received) → submitted

How CMO Submits Data per Batch

From the Batch Detail page, the CMO has two options depending on their active connections:

  • SFTP: Upload files to the path shown on screen: /{batchId}/{elementType}/{filename}. The system auto-detects the element type and marks it as received.
  • AI Upload: Click the "Upload" button next to an element → select a PDF or image → AI processes and marks element received.
  • Manual mark: Click "Mark Received" for elements submitted through other channels.

CoA Document Viewer

When a Certificate of Analysis is uploaded via the AI pattern, the extracted content is viewable directly in the portal:

  1. Navigate to Batch Detail → find the CoA element with status "Received"
  2. Click the View button
  3. A modal opens showing:
  • Confidence badge — ≥85% (auto-processed) or <85% (manual review needed)
  • Document header: Product Name, Batch/Lot Number, Manufacturing Date, Expiry Date, Quantity, Disposition, Analyst, QA Approver
  • Test Results table: Each quality test with Method, Specification, Result, Unit, and PASS/FAIL status

SLA Tracking & Compliance

Each contract defines per-element SLA deadlines (e.g., "CoA must arrive within 5 days of manufacturing date"). The platform actively monitors compliance:

How SLA Tracking Works

  1. Contract specifies elementSlas: e.g., {"elementType": "CoA", "maxDaysAfterBatch": 5}
  2. EventBridge triggers the SLA Checker Lambda daily at 6am UTC
  3. For each in-progress batch: compares manufacturingDate + maxDaysAfterBatch vs. today
  4. Unreceived elements past deadline are flagged as overdue
  5. Dashboard shows overdue counts, visual indicators, and filter by overdue status

Dashboard Features

  • Summary cards: Active Contracts, Total Batches, Incomplete, Overdue
  • Run SLA Check: Manual trigger for immediate check (in addition to daily auto-run)
  • Filters: Filter by CMO, by status (has overdue elements, has missing elements)
  • Click-through: Click any batch row to navigate directly to its detail page

CMO Detail Page

Clicking a CMO name in CMO Partners opens a comprehensive detail page with 4 tabs:

  • Connections — all connections for this CMO (create, activate, view credentials)
  • Schemas — all schemas defined for this CMO (create, expand, register)
  • Data Contracts — all contracts for this CMO with status
  • Batches — all batch submissions for this CMO with completeness

This provides a single-pane view of everything related to a specific CMO without navigating multiple pages.

Multi-Tenant Data Isolation

The platform enforces strict data isolation between CMOs:

  • CMO user accounts carry a custom:organization claim in their JWT token
  • All API endpoints enforce filtering server-side using this JWT claim
  • CMO users cannot see data from other organizations — even if they modify query parameters
  • Merck admins see all data across all CMOs (with optional CMO filter)
  • S3 data lake paths are partitioned by cmoId for physical isolation

Natural Language Query

The NL Query page allows Merck admins to ask questions about CMO data in plain English. Powered by Amazon Bedrock, it translates natural language into SQL queries against the data lake via Amazon Athena.

Example queries:

  • "Show me all batches from Lonza that are overdue"
  • "What is the average yield for Ibuprofen 200mg in the last 30 days?"
  • "Which CMOs have missing CoA documents?"
